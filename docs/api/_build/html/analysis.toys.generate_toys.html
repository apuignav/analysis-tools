generate_toys¶

Toy generation.

analysis.toys.generate_toys.generate(physics_factory, n_events)[source]¶

Perform generation of toys.

Note:
If the factory is simultaneous, events are generated in steps. For that reason, the configuration for ‘gen/nevents’ must be a dictionary of {label -> nevents} keys.
Arguments:
physics_factory (analysis.physics.PhysicsFactory): Physics factory object to get
observables, parameters and PDFs from.

n_events (dict, int): Number of events to generate.

Return:
pandas.DataFrame: Generated events.
Raise:
ValueError: If the number of events to generate is not properly specified. KeyError: If an unknown simultaneous category label is requested.
analysis.toys.generate_toys.main()[source]¶

Toy generation application.

Parses the command line and runs the toys, catching intermediate errors and transforming them to status codes.

Status codes:
0: All good. 1: Error in the configuration files. 2: Files missing (configuration or toys). 3: Error configuring physics factories. 4: Error in the event generation. 128: Uncaught error. An exception is logged.
analysis.toys.generate_toys.run(config_files, link_from)[source]¶

Run the script.

Arguments:
config_files (list[str]): Path to the configuration files. link_from (str): Path to link the results from.
Raise:

KeyError: If some configuration data are missing. OSError: If there either the configuration file does not exist or if

there is a problem preparing the output path.

ValueError: If there is any problem in configuring the PDF factories. RuntimeError: If there is a problem during the generation.
